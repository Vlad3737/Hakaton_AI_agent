# Архитектура агента для анализа научных статей

## Обзор

Система реализует ИИ-агента для анализа научных статей на базе **LangGraph** и **GigaChat-2-Max**. Агент может отвечать на вопросы по статьям, выполняя:
- Извлечение текста из PDF и LaTeX источников
- Извлечение и анализ изображений (рисунков)
- Использование RAG (Retrieval-Augmented Generation) для контекстных ответов
- Обработку мультимедийного контента (текст + рисунки)

## Детали компонентов

### 1. Состояние агента (`AgentState`)

**Расположение:** `src/agent/graph.py`

Состояние LangGraph на основе `MessagesState`:

| Поле | Тип | Описание |
|------|-----|----------|
| `question_id` | `int` | Номер вопроса |
| `question` | `str` | Текст исходного вопроса |
| `answer` | `str` | Сгенерированный ответ |
| `search_results` | `Annotated[list[str], _reduce_search_results]` | Извлечённые контекстные фрагменты |
| `images_needed` | `list[str]` | Список имен изображений, на которые ссылается вопрос |
| `reasoning_steps` | `Annotated[list[str], _reduce_reasoning_steps]` | Трасса рассуждений агента |
| `time_taken` | `float` | Время обработки в секундах |
| `completed` | `bool` | Статус завершения обработки |

### 2. Узлы графа

| Узел | Функция | Описание |
|------|---------|----------|
| `parse_question` | `parse_question()` | Анализирует вопрос для определения стратегии (текстовый или мультимодальный) |
| `analyze_images` | `analyze_image_step()` | Обрабатывает ссылочные изображения через MultimodalQuery |
| `search_knowledge` | `search_knowledge()` | Извлекает релевантные текстовые чанки через RAGSystem |
| `generate_answer` | `generate_answer()` | Объединяет контекст и генерирует финальный ответ |
| `format_answer` | `format_final_answer()` | Форматирует вывод с номером вопроса |

### 3. RAGSystem

**Расположение:** `src/agent/components.py`
[text](src/agent/graph.py)
**Возможности:**
- Загружает PDF файлы (извлекает текст + изображения)
- Загружает LaTeX документы (извлекает текст + рисунки + уравнения)
- Встраивает содержимое в ChromaDB
- Поддерживает гибридный поиск: текстовый поиск + поиск по метаданным изображений

**Основные методы:**
- `load_and_chunk()` - Загрузка и чанкинг всех источников
- `retrieve(query, n_results)` - Извлечение текстовых чанков
- `retrieve_images(image_name, n_results)` - Поиск конкретных изображений

### 4. Обработчики запросов

**GigaChatQuery:**
- Ответы на вопросы по тексту
- Использует GigaChat-2-Max с temperature=0.2

**MultimodalQuery:**
- Анализ изображений через GigaChat-2-Max
- Преобразует изображения в base64 для мультимодальной обработки
- Обеспечивает контекстные описания изображений

### 5. Загрузчики данных

**PDFLoader:**
- Использует PyMuPDF для извлечения текста
- Извлекает изображения из страниц PDF
- Чанкинг текста для RAG

**TexLoader:**
- Парсит главный LaTeX файл
- Обнаруживает и извлекает файлы рисунков
- Извлекает уравнения из LaTeX источника

**ImageLoader:**
- Загружает изображения через PIL
- Преобразует в base64 для API
- Хранит метаданные изображений

## Ключевые технологии

| Библиотека | Назначение |
|------------|------------|
| **LangGraph** | Модель конечного автомата для рабочего процесса агента |
| **LangChain** | Интеграция с GigaChat, RAG |
| **langchain-gigachat** | Интеграция модели GigaChat-2-Max |
| **ChromaDB** | Векторная база данных для RAG |
| **PyMuPDF** | Извлечение текста и изображений из PDF |
| **PIL/Pillow** | Обработка изображений |
| **python-dotenv** | Управление переменными окружения |

## Конфигурация

**Переменные окружения (`.env`):**
```bash
GIGACHAT_CREDENTIALS='<ваши-учетные-данные>'
GIGACHAT_SCOPE='GIGACHAT_API_CORP'
```

**Настройки модели:**
- Модель: `GigaChat-2-Max`
- Температура: `0.2`
- Таймаут: `120` секунд
- Проверка SSL: `False` (для корпоративной сети)

---

## Соображения производительности

1. **Параллельная обработка:** Изображения и текстовый поиск выполняются параллельно (ребра LangGraph)
2. **Пакетная загрузка:** Весь PDF и LaTeX контент загружается один раз, а не по каждому вопросу
3. **Эмбеддинги:** all-MiniLM-L6-v2 для эффективного векторного поиска
4. **Размер чанка:** 512 токенов с перекрытием 128
5. **Кэширование:** ChromaDB кэшируется в `.cache/chroma/`

---

## Обработка ошибок

- **Ошибка 400 OAuth:** Некорректное значение `GIGACHAT_SCOPE` в `.env`
- **Таймаут:** Увеличьте `timeout` в конструкторе `GigaChat()`
- **Обрезка ответов:** Ответы >5000 символов обрезаются с предупреждением
